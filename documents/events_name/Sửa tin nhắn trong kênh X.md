# "Sửa tin nhắn trong kênh X"

X is replaced by the channel name. 3 types: voice channel, text channel, mod only channel

Full name examples:
- `"Sửa tin nhắn trong kênh Phòng Học"` (voice channel)
- `"Sửa tin nhắn trong kênh tin-tuc"` (text channel)
- `"Sửa tin nhắn trong kênh huong-dan"` (mod-tracked channel)
- `"Sửa tin nhắn trong kênh luat-server"` (mod-tracked channel)
- `"Sửa tin nhắn trong kênh kien-thuc-marketing"` (mod-tracked channel)

Note: mod-tracked channels produce this event name via the generic text channel
pattern. A separate mod update event is NOT generated by this scenario —
ModMessageHandler only fires its own event when `author_id` matches a
configured moderator ID. If the editor is not the designated moderator, only
this generic edit event fires.

## Trigger Condition

Fired when a `message_edited` event occurs in a voice or text channel.

---

## Collection Logic

**GenericChannelProcessor** resolves the pattern `"Sửa tin nhắn trong kênh {channel_name}"`.
For voice channels, category-based classification applies (fixed vs custom).
For text channels, the `author_id` is passed in event data but only
**ModMessageHandler** uses it to check moderator identity. If the author is a
registered moderator in that channel, ModMessageHandler fires a separate named
event; otherwise only this generic edit pattern fires.
